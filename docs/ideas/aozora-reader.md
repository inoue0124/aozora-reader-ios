# 青空文庫リーダー アイデアメモ（壁打ち反映）

## コンセプト
「青空文庫の中身は最高、見た目は現代的に」な iOS 読書アプリ。

既存の青空文庫体験はタイトル・著者名中心で殺風景なので、
このアプリでは **発見体験** と **読書体験** の両方を強化する。

## 方向性（A/B/C 全採用）
- A: 読む体験特化（読書継続しやすさ）
- B: 作品発見特化（探しやすさ・出会いやすさ）
- C: 青空文庫の難所対応（ルビ・注記・外字）

## 主要機能（MVP + 早期拡張）

### 1) 読書コア
- 本文表示（縦スクロール、将来は縦書きも検討）
- しおり（読書位置保存）
- お気に入り
- 読書設定（フォントサイズ、行間、余白、テーマ）
- ルビ・注記・外字の読みやすい表示

### 2) 発見コア
- 作品検索（タイトル、著者）
- 著者詳細ページ
  - 著者プロフィール
  - **著者の顔画像表示**（権利/出典に注意）
  - 代表作一覧
- 作品詳細ページ
  - **AI 生成の表紙画像**を表示して一覧を華やかに
  - ただし出典誤認を防ぐため「参考ビジュアル」表記を検討

### 3) 体験拡張
- **オフライン読書**（作品本文キャッシュ）
- レビュー機能（作品への感想・評価）
  - MVPではローカルメモ形式でも可
  - 将来的に共有レビューへ拡張

## 非機能要件メモ
- iOS 17+、SwiftUI + MVVM
- 起動・スクロールは軽快
- 広告は付けない（広告収益モデルなし）

## 要検討（重要）
- 青空文庫データ取得元（API / 公式データ）
- 著者画像の権利と出典（Wikipedia/Wikimedia等の利用条件）
- AI表紙生成の運用
  - 生成コスト
  - 著作権・スタイル模倣リスク
  - キャッシュ戦略
- レビュー機能のモデレーション方針（公開する場合）

## 最初の実装優先案
1. 検索 → 作品詳細 → 読書 → しおり
2. 著者詳細（顔画像付き）
3. AI表紙（まずは非同期生成 + キャッシュ）
4. オフライン読書
5. レビュー（ローカル）
