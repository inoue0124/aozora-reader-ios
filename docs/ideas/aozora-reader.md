# 青空文庫リーダー v2 アイデアメモ（縦書きファースト / 発見体験重視）

## プロダクト方針
- ゼロから再構築する
- 読書体験は **縦書き・ページめくりファースト**
- ホームは「検索起点」ではなく「発見起点」
- 他社電子書籍アプリのような「棚（レールUI）」で回遊させる
- 広告はなし

## 体験コンセプト
「作品を探すアプリ」ではなく、
「今日読みたい1冊に自然に出会えるアプリ」にする。

## 主要UX（MVP）

### 1. ホーム（発見画面）
- おすすめ著者カルーセル棚
- 最近のレビュー棚
- 作品タイプ別の棚（短編 / 長編 / エッセイ / 戯曲 など）
- 続きから読むカード（しおり復帰）
- 新着・注目作品棚（データ更新設計に応じて）

### 2. 作品詳細
- タイトル / 著者 / 出典
- **あらすじ（生成 or 既存情報由来）**
- 表紙（AI生成はプレースホルダー可）
- レビュー一覧（ローカル中心）
- 「読む」導線を最優先

### 3. リーダー
- **縦書きページめくりがデフォルト**
- 横書きスクロールは切替オプション
- ページ位置表示（現在/総ページ）
- 読書設定（フォント、行間、余白、テーマ）
- しおり保存・復帰
- オフライン読書（キャッシュ）

### 4. レビュー
- ローカル保存レビュー（星 + コメント）
- ホームに最近レビューを棚表示
- 将来の共有レビュー拡張を見据えたモデル設計

## 技術メモ
- SwiftUI + MVVM + SwiftData
- 縦書きは WebView + CSS writing-mode 方式をベースに改善
- 作品メタはバンドルJSONを起点、将来API差し替え可能に

## 検討事項（先に決めたい）
- 作品タイプ分類の定義ルール
- おすすめ著者の選定ロジック（固定 / スコア / 行動履歴）
- あらすじ生成のソースと品質担保
- 棚の並び順ロジック（パーソナライズ有無）

## 実装優先順
1. ホーム棚UI基盤（カルーセル + レール）
2. 作品詳細の情報密度アップ（あらすじ含む）
3. 縦書きリーダー品質改善（ページ体験）
4. 棚データ連動（おすすめ著者 / 最近レビュー / 作品タイプ）
5. 調整・チューニング
